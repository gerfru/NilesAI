# Phase 2: n8n Docker Installation & Setup

## Übersicht

Installation von n8n als self-hosted Workflow-Automation-Platform via Docker Compose.

n8n läuft zusammen mit Evolution API und PostgreSQL in einem gemeinsamen Docker-Netzwerk (`niles_network`).

## Voraussetzungen

- Docker Desktop installiert
- LM Studio läuft auf Port 1234

## Schritte

### 1. Docker installieren (falls nicht vorhanden)

```bash
# Docker Desktop für Mac
brew install --cask docker

# Docker starten
open -a Docker

# Warte bis Docker läuft, dann testen:
docker ps
```

### 2. Environment konfigurieren

```bash
cd ~/Documents/Niles

# .env aus Template erstellen
cp .env.example .env

# Werte anpassen
nano .env
```

Benötigte Variablen in `.env`:

- `EVOLUTION_API_KEY` — API Key für Evolution API + n8n Workflows
- `EVOLUTION_POSTGRES_PASSWORD` — PostgreSQL Passwort
- `N8N_API_KEY` — n8n API Key (optional, für import-workflows.sh)

### 3. Alle Services starten

```bash
./scripts/start.sh

# Oder manuell:
docker compose -f docker/docker-compose.yml --env-file .env up -d

# Status prüfen
docker ps
```

Das startet:

- **n8n** auf Port 5678
- **Evolution API** auf Port 8080
- **PostgreSQL** auf Port 5432

### 4. n8n öffnen & User anlegen

1. Browser: `http://localhost:5678`
2. Account erstellen (läuft lokal, keine Cloud!)
   - Email: Beliebig (wird nur lokal gespeichert)
   - Passwort: Sicher wählen

### 5. LM Studio als Credential konfigurieren

**In n8n:**

1. Settings → Credentials
2. "+ Add Credential"
3. Suche: "OpenAI"
4. Konfiguration:
   - **API Base URL:** `http://host.docker.internal:1234/v1`
   - **API Key:** `lm-studio` (beliebig, wird nicht geprüft)
   - Organization ID: Leer lassen
5. "Test Connection" → Sollte "Connection successful" zeigen
6. Save

**WICHTIG:**

- Credential-Typ heißt "OpenAI" aber wir nutzen LM Studio (lokal!)
- `host.docker.internal` = Docker-Zugriff auf localhost
- Keine echten OpenAI API Keys erforderlich

### 6. PostgreSQL Credential konfigurieren

**In n8n:**

1. Settings → Credentials
2. "+ Add Credential"
3. Suche: "Postgres"
4. Konfiguration:
   - **Host:** `evolution_postgres` (Container-Name im Docker-Netzwerk)
   - **Database:** `evolution_db`
   - **User:** `evolution`
   - **Password:** (dein `EVOLUTION_POSTGRES_PASSWORD` aus `.env`)
   - **Port:** `5432`
5. Save

Wird verwendet für den Kontakt-Sync und Contact-Lookup im WhatsApp-Workflow.

### 7. Test-Workflow erstellen

**Erster AI Agent Workflow:**

1. New Workflow
2. Node: "When chat message received"
3. Node: "AI Agent"
   - Chat Model: OpenAI Chat Model (dein LM Studio Credential)
   - Memory: Simple Memory
   - **WICHTIG:** "Use Response API" deaktivieren!
4. Save & Test

## Troubleshooting

### Container startet nicht

```bash
# Logs prüfen
docker logs niles_n8n

# Port bereits belegt?
lsof -i :5678

# Container neu starten
docker compose -f docker/docker-compose.yml restart n8n
```

### "Connection failed" bei LM Studio Credential

- LM Studio Server läuft? Check: `curl http://localhost:1234/v1/models`
- Base URL korrekt: `http://host.docker.internal:1234/v1` (NICHT localhost!)

### AI Agent Workflow Error: "Bad request - please check your parameters"

- **"Use Response API" deaktivieren** in AI Agent Settings!
